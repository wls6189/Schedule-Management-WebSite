# 스케줄 관리 웹사이트

3명의 사용자가 작업 날짜와 시간에 맞춰 출석을 관리하는 웹 애플리케이션입니다.

## 주요 기능

1. **요일 선택**: 월요일부터 일요일까지 요일 선택 가능
2. **시간 선택**: 시작 시간과 종료 시간 설정 (예: 09시~15시, 10시~17시)
3. **출석 체크**: 출석 완료/미완료 상태 표시 및 관리

## 설치 및 실행

```bash
# 의존성 설치
npm install

# 서버 실행
npm start

# 개발 모드 (nodemon 사용)
npm run dev
```

서버는 기본적으로 `http://localhost:3000`에서 실행됩니다.

## 네트워크 접근 설정 (다른 사용자와 공유)

### 같은 네트워크(Wi-Fi/이더넷)에서 접근하기

1. **이 컴퓨터의 IP 주소 확인**
   - Windows: `ipconfig` 명령어 실행 후 "IPv4 주소" 확인
   - 예: `192.168.0.100`

2. **서버 실행**
   ```bash
   npm start
   ```

3. **다른 사용자에게 주소 공유**
   - `http://[이 컴퓨터의 IP 주소]:3000`
   - 예: `http://192.168.0.100:3000`

4. **방화벽 설정** (필요한 경우)
   - Windows 방화벽에서 포트 3000 인바운드 규칙 추가
   - 또는 방화벽 경고가 나타나면 "액세스 허용" 클릭

### 주의사항
- 같은 Wi-Fi 네트워크 또는 이더넷에 연결되어 있어야 합니다
- 공용 Wi-Fi에서는 보안상 접근이 제한될 수 있습니다

## 🌐 인터넷 접근 설정 (어디서나 접근 가능)

### 방법 1: ngrok 사용 (빠른 테스트용) ⚡

**가장 빠르고 간단한 방법입니다!**

1. **ngrok 계정 생성 및 토큰 받기**
   - https://dashboard.ngrok.com/signup 에서 무료 계정 생성
   - 인증 토큰 복사

2. **환경 변수 설정** (선택사항)
   ```bash
   # Windows PowerShell
   $env:NGROK_AUTHTOKEN="당신의_토큰"
   
   # 또는 .env 파일 생성 (권장)
   ```

3. **서버 실행** (첫 번째 터미널)
   ```bash
   npm start
   ```

4. **ngrok 터널 시작** (두 번째 터미널)
   ```bash
   npm run tunnel
   ```
   또는 수동으로:
   ```bash
   npx ngrok http 3000
   ```

5. **공유할 주소**
   - ngrok이 생성한 `https://xxxx-xxxx.ngrok-free.app` 주소를 공유
   - 이 주소는 인터넷 어디서나 접근 가능합니다!

**주의**: 무료 플랜은 세션이 2시간 후 종료됩니다.

### 방법 2: 클라우드 배포 (영구적) ☁️

**영구적인 인터넷 접근이 필요하면 클라우드에 배포하세요!**

#### Render 배포 (추천 - 가장 간단)
- 자세한 방법: `deploy-render.md` 파일 참조
- 무료 플랜 제공
- GitHub 연동으로 자동 배포

#### Railway 배포
- 자세한 방법: `deploy-railway.md` 파일 참조
- 무료 플랜 제공
- 간단한 설정

**배포 후 얻는 것:**
- ✅ 영구적인 인터넷 주소
- ✅ HTTPS 자동 설정
- ✅ 자동 배포 (GitHub push 시)
- ✅ 언제 어디서나 접근 가능

## 사용 방법

1. 웹 브라우저에서 `http://localhost:3000` (또는 네트워크 IP 주소) 접속
2. 사용자 이름을 입력하여 로그인
3. 요일과 시간을 선택하여 스케줄 등록
4. 출석 완료 여부를 체크하여 관리
