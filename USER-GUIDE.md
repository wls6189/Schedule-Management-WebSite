# 사용자 가이드

## 👥 다른 사용자가 사용하는 방법

**중요**: 다른 사용자는 터미널이 필요 없습니다! 브라우저만 있으면 됩니다.

### 단계별 사용 방법

1. **브라우저에서 주소 접속**
   - Render 배포 시: `https://your-app.onrender.com`
   - 로컬 네트워크: `http://192.168.x.x:3000`
   - ngrok 사용 시: `https://xxxx-xxxx.ngrok-free.app`

2. **로그인하기**
   - 화면 상단의 "사용자 이름 입력" 필드에 자신의 이름 입력
   - 예: "홍길동", "김철수", "이영희"
   - "로그인" 버튼 클릭
   - ✅ 완료! 이제 스케줄을 추가할 수 있습니다

3. **스케줄 추가하기**
   - 요일 선택 (월~일)
   - 시작 시간 선택
   - 종료 시간 선택
   - "스케줄 추가" 버튼 클릭

4. **출석 체크하기**
   - 스케줄 목록에서 출석 체크박스 클릭
   - ✅ 출석 완료 / ⏳ 출석 대기 상태 변경

### 주의사항

- ✅ **터미널 불필요**: 다른 사용자는 브라우저만 있으면 됩니다
- ✅ **별도 설치 불필요**: 웹 브라우저만 있으면 사용 가능합니다
- ✅ **동시 사용 가능**: 여러 사용자가 동시에 사용할 수 있습니다
- ✅ **자동 동기화**: 다른 사용자가 추가한 스케줄이 자동으로 표시됩니다

---

## 🖥️ 서버 관리자 (당신)가 해야 할 일

### 처음 한 번만 설정

1. **서버 실행** (터미널 필요)
   ```bash
   npm start
   ```

2. **Render에 배포** (한 번만)
   - Render 대시보드에서 배포 완료
   - 이후 자동으로 배포됩니다

### 일상적인 사용

- **로컬 서버 사용 시**: 서버를 실행해두면 됩니다
- **Render 배포 시**: 별도 작업 없이 자동으로 실행됩니다

## ❓ 자주 묻는 질문

**Q: 다른 사용자도 터미널을 설치해야 하나요?**
A: 아닙니다! 브라우저만 있으면 됩니다.

**Q: 다른 사용자도 Node.js를 설치해야 하나요?**
A: 아닙니다! 서버 관리자만 필요합니다.

**Q: 다른 사용자가 코드를 수정할 수 있나요?**
A: 웹사이트에서 스케줄만 추가/수정/삭제할 수 있습니다. 코드 수정은 불가능합니다.

**Q: 동시에 몇 명까지 사용할 수 있나요?**
A: 제한 없습니다! 여러 명이 동시에 사용할 수 있습니다.
