# 🌐 다른 사용자가 웹사이트에 접속하는 방법

## ⚠️ 매우 중요한 안내

**다른 사용자는 다음이 전혀 필요 없습니다:**
- ❌ `npm start` 명령어 입력
- ❌ 터미널 사용
- ❌ Node.js 설치
- ❌ 코드 다운로드
- ❌ GitHub 계정
- ❌ 어떤 프로그램 설치도 불필요

**필요한 것:**
- ✅ 웹 브라우저만 있으면 됩니다! (Chrome, Edge, Firefox, Safari 등)

**💡 무료 플랜에서도 가능합니다!**
- ✅ Render 무료 플랜이어도 다른 사용자들이 접속할 수 있습니다
- ✅ 접속자 수 제한 없습니다
- ✅ 동시 접속 가능합니다
- ✅ 인터넷 어디서나 접속 가능합니다

---

## 📋 접속 방법 (2가지)

### 방법 1: Render 배포 주소 사용 (인터넷 어디서나 접속 가능) ⭐ 추천

#### 전제 조건
- 서버 관리자가 Render에 배포를 완료했어야 합니다
- Render에서 제공받은 주소가 있어야 합니다
- 예: `https://schedule-management.onrender.com`

#### 단계별 접속 방법

**1단계: 웹 브라우저 열기**
- Chrome, Edge, Firefox, Safari 등 어떤 브라우저든 가능합니다
- 스마트폰 브라우저도 가능합니다

**2단계: 주소창에 URL 입력**
- 브라우저 상단의 주소창(URL 바)을 클릭합니다
- 서버 관리자에게 받은 주소를 입력합니다
  - 예: `https://schedule-management.onrender.com`
- Enter 키를 누릅니다

**3단계: 웹사이트 로딩 대기**
- 처음 접속 시 약 30초~1분 정도 걸릴 수 있습니다 (슬리프 모드에서 깨어나는 시간)
- "Loading..." 또는 "Starting..." 메시지가 보일 수 있습니다
- 조금만 기다리면 웹사이트가 나타납니다

**4단계: 로그인하기**
- 화면 상단에 "사용자 이름 입력" 필드가 보입니다
- 자신의 이름을 입력합니다
  - 예: "홍길동", "김철수", "이영희", "박영수" 등
- "로그인" 버튼을 클릭합니다
- 로그인 성공 시 "현재 사용자: [이름]"이 표시됩니다

**5단계: 사용하기**
- 이제 스케줄을 추가하고 출석을 체크할 수 있습니다!

---

### 방법 2: 로컬 네트워크 주소 사용 (같은 Wi-Fi/이더넷 필요)

#### 전제 조건
- 서버 관리자와 **같은 Wi-Fi 네트워크** 또는 **같은 이더넷**에 연결되어 있어야 합니다
- 서버 관리자가 서버를 실행 중이어야 합니다
- 서버 관리자에게 IP 주소를 받아야 합니다
  - 예: `http://192.168.219.103:3000`

#### 단계별 접속 방법

**1단계: 같은 네트워크에 연결**
- 서버 관리자와 같은 Wi-Fi에 연결되어 있는지 확인합니다
- 또는 같은 이더넷 케이블에 연결되어 있는지 확인합니다

**2단계: 웹 브라우저 열기**
- Chrome, Edge, Firefox 등 어떤 브라우저든 가능합니다

**3단계: 주소창에 IP 주소 입력**
- 브라우저 주소창에 서버 관리자에게 받은 주소를 입력합니다
  - 예: `http://192.168.219.103:3000`
- **주의**: `http://`를 반드시 포함해야 합니다
- Enter 키를 누릅니다

**4단계: 보안 경고 처리 (처음 한 번만)**
- Windows 방화벽 경고가 나타날 수 있습니다
- "액세스 허용" 또는 "Allow access"를 클릭합니다
- 또는 서버 관리자에게 방화벽 설정을 요청합니다

**5단계: 로그인하기**
- 화면 상단에 "사용자 이름 입력" 필드가 보입니다
- 자신의 이름을 입력합니다
- "로그인" 버튼을 클릭합니다

**6단계: 사용하기**
- 이제 스케줄을 추가하고 출석을 체크할 수 있습니다!

## 📱 스마트폰에서 접속하는 방법

### Android / iPhone 모두 가능합니다!

**1단계: 브라우저 앱 열기**
- Chrome, Safari, Samsung Internet 등 어떤 브라우저든 가능합니다

**2단계: 주소 입력**
- 주소창을 탭합니다
- 서버 관리자에게 받은 주소를 입력합니다
  - Render: `https://schedule-management.onrender.com`
  - 로컬: `http://192.168.219.103:3000`
- "이동" 또는 "Go" 버튼을 탭합니다

**3단계: 로그인 및 사용**
- 데스크톱과 동일하게 사용할 수 있습니다!

## ❓ 자주 묻는 질문 (FAQ)

### Q1: 다른 사용자도 `npm start`를 입력해야 하나요?
**A: 절대 아닙니다!** `npm start`는 서버 관리자만 실행하면 됩니다.  
다른 사용자는 브라우저에서 주소만 입력하고 로그인하면 됩니다.

### Q2: 다른 사용자도 Node.js를 설치해야 하나요?
**A: 아닙니다!** 다른 사용자는 Node.js, npm, 터미널 등이 전혀 필요 없습니다.  
웹 브라우저만 있으면 됩니다.

### Q3: 다른 사용자도 코드를 다운로드해야 하나요?
**A: 아닙니다!** 코드 다운로드나 설치가 필요 없습니다.  
브라우저에서 주소만 입력하면 바로 사용할 수 있습니다.

### Q4: 다른 사용자도 GitHub 계정이 필요하나요?
**A: 아닙니다!** GitHub 계정이 필요 없습니다.  
웹사이트 주소만 있으면 누구나 접속할 수 있습니다.

### Q5: 로그인할 때 비밀번호가 필요한가요?
**A: 아닙니다!** 비밀번호 없이 이름만 입력하면 됩니다.  
예: "홍길동" 입력 → 로그인 버튼 클릭 → 완료!

### Q6: 여러 명이 동시에 사용할 수 있나요?
**A: 네!** 여러 명이 동시에 접속해서 사용할 수 있습니다.  
각자 자신의 이름으로 로그인하면 됩니다.

### Q7: 다른 사용자가 추가한 스케줄을 볼 수 있나요?
**A: 네!** 모든 사용자가 추가한 스케줄이 자동으로 표시됩니다.  
5초마다 자동으로 새로고침되어 다른 사용자의 변경사항이 반영됩니다.

### Q8: 접속이 안 되면 어떻게 하나요?
**A: 다음을 확인하세요:**
- 주소가 정확한지 확인 (오타 없는지)
- Render 배포: 서버 관리자가 배포를 완료했는지 확인
- 로컬 네트워크: 같은 Wi-Fi에 연결되어 있는지 확인
- 서버 관리자에게 문의

### Q9: 스마트폰에서도 사용할 수 있나요?
**A: 네!** 스마트폰 브라우저에서도 완벽하게 작동합니다.  
데스크톱과 동일하게 사용할 수 있습니다.

### Q10: 서버 관리자는 무엇을 해야 하나요?
**A: 서버 관리자만 처음 한 번 배포하면 됩니다:**
- Render에 배포 (한 번만)
- 또는 로컬 서버 실행 (`npm start` - 서버 관리자만!)
- 이후 자동으로 실행되므로 다른 사용자는 아무것도 할 필요 없습니다

## 📞 서버 관리자에게 요청할 정보

다른 사용자가 접속하려면 서버 관리자에게 다음 정보를 요청하세요:

### Render 배포 시:
- ✅ 웹사이트 주소 (예: `https://schedule-management.onrender.com`)

### 로컬 네트워크 시:
- ✅ 서버 IP 주소 (예: `192.168.219.103`)
- ✅ 포트 번호 (보통 `3000`)
- ✅ 전체 주소 (예: `http://192.168.219.103:3000`)

다른 사용자가 접속하기 전에 확인:

- [ ] 주소를 정확히 입력했나요? (`http://` 또는 `https://` 포함)

- [ ] 로컬 네트워크: 같은 Wi-Fi에 연결되어 있나요?

**모든 항목이 체크되면 접속할 수 있습니다!** 🎉
